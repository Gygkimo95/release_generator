## 迭代过程摘要

本次文档从 AI 初版到定稿共经历了 **3 轮反馈迭代**，涵盖内容定位、用户视角和文风调性等维度。

### 反馈记录

| 轮次 | 反馈类型 | 问题 | 调整措施 |
|---|---|---|---|
| 1 | content | 自定义模型测评暗示必须全选维度；自定义数据集测评暗示必须使用全部模型 | 明确强调"按需选择"，用具体场景举例不同选择策略 |
| 2 | style | 全文以平台视角罗列功能（"我们提供了X"），缺乏用户代入感 | 重构为用户视角：适用场景→获得的价值→操作流程；摘要改为从用户痛点切入 |
| 3 | style | 发版文档中"您"出现频率过高，不符合专业文档调性 | 省略主语或改用"用户"等中性表达，控制第二人称使用频率 |

## 经验提炼结果

### 新增规则（5条）

| ID | 类别 | 规则 |
|---|---|---|
| r025 | style | 功能发版文档应从用户视角撰写（谁在用、什么场景、解决什么问题、获得什么价值），而非平台视角 |
| r026 | style | 发版文档中避免过多使用第二人称"您"，应使用省略主语、"用户"等中性表达 |
| r027 | content | 描述可选功能时必须明确"按需选择"，用具体场景举例不同选择策略 |
| r028 | structure | 功能发版文档摘要应从用户痛点切入，再引出功能如何回应需求 |
| r029 | formatting | 功能对比表应以用户场景为行维度，列维度为"推荐模式→需要准备→将获得" |

### 新增反模式（2条）

### 新增 Prompt 补丁（2条）

| ID | 内容 |
|---|---|

## 总结

本次迭代的核心经验归纳为三大方面：

1. **视角转换**：功能发版文档应站在用户立场，以"谁在用→什么场景→获得什么"的结构组织，而非平台视角的功能罗列
2. **灵活性传达**：可选配置项必须明确"按需选择"的定位，用具体场景举例说明不同选择策略，避免暗示全部使用
3. **文风调性**：发版文档保持专业中性语气，控制第二人称使用频率，用省略主语等方式替代
